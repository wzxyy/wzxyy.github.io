Vue.js 基础语法

指令是什么：由Vue提供的特殊的html行内属性

1.v-text 指令

  • 作用：替换掉原来所有的内容，
    • 类似于 innerText

2.v-html指令

  • 作用：替换掉原来所有的内容，遇到标签会解析成DOM元素，主要用在富文本
    • 类似于 innerHTML

3.v-model指令

  • 作用：双向绑定值，输入的内容与数据的改变相互影响
    • 基本用于表单元素（input,textarea,select）
    • 可以扩展到组件上（此时仅作了解）

  • 原理：原生input事件

    • $event 指代当前触发的事件对象。

    • $event.target 指代当前触发的事件对象的dom

    • $event.target.value 就是当前dom的value值

3.1 v-model.lazy

  • 作用：v-model 在每次 input 事件触发后会同步数据，添加 lazy 修饰符,将转为在 change 事件之后进行同步，也可理解为失去焦点后

4.v-on指令

  • 作用：绑定事件
    • 推荐使用简写形式 @

4.1 @.stop

  • 作用：阻止事件冒泡
  • 类型：v-on修饰符

4.2 @.prevent

  • 作用：阻止事件默认行为
  • 类型：v-on修饰符

4.3 @.keyup.xx

  • 作用：按下键盘 xx 键，触发，
    • xx 可以是 keycode，
    • xx 也可以是 a,enter,F1 等
  • 类型：v-on修饰符

4.4 @.self

  • 作用：只当在 event.target 是当前元素自身时触发处理函数
    • 即事件不是从内部元素触发的
  • 类型：v-on修饰符

5.v-bind指令

  • 作用：data属性值 绑定行内属性
    • 可以绑定 所有的行内属性 包括 行内自定义属性
    • 主要用在 class 和 style 属性里
    • 推荐简写形式： 直接冒号

6.v-for指令

  • 作用：遍历数组或者对象，生成对应的标签
    • key就是给标签做一个唯一标识
    • 语法检查工具下，v-for不写 :key 会报错
  • key的特殊作用
    • 当在input内输入值并且切换的时候，复用的效果很明显
    • vue会从性能的角度去考虑复用，频繁的操作DOM（特别是移除和创建新节点会比较耗性能）
    • 给元素一个唯一标识key，那么复用的时候就不会出错
    • 但是一般不用key也行，当我们写上 v-model 后这个问题也会被解决
  • 数据双向改变注意点：
    • 修改数组或对象的数据（复杂类型数据），界面也会跟着发生改变
    • 基本类型不会跟随改变

7.v-if-else指令

  • 作用：按条件决定是否渲染元素
    • v-if可以单独使用
    • v-if和v-else-if以及v-else的搭配必须是同级的兄弟关系

8. v-show指令

  • 作用：控制元素显示还是隐藏
    • 它仅仅只是改display的值为block或者none
    • 如果需要频繁的显示隐藏，用v-show性能更高
    • 实际开发v-if用的相对更多一些

9.v-cloak指令

  • 作用：控制vue实例化完成前的dom样式，vue加载前的一些状态
    • v-cloak 相当于设置一个类名，然后自定义样式
    • vue做的事只是加载后把v-cloak删掉

10.v-once指令

  • 作用：只渲染元素和组件一次。随后的重新渲染，元素/组件及其所有的子节点将被视为静态内容并跳过。这可以用于优化更新性能。

11.v-pre指令

  • 作用：跳过这个元素和它的子元素的编译过程。可以用来显示原始 Mustache 标签。跳过大量没有指令的节点会加快编译。

12.ref-获取dom

  • 作用：Vue里面专门用来获取dom元素的
    • 在需要找的标签行内写 ref=”name”
    • 获取此dom：this.$refs.name
    • name 可以一样，但只会拿到最后一个

13.单元素动画

  • 语法：
    • 用 transition 把需要动画的元素包起来
    • 给transition一个name属性，指定动画效果
  • 注意：
    • 执行完动画后，进行动画的元素会被删除掉

13.1结合@keyframes

  • 写一个@keyframes的动画剧本

  • 放在 xx-enter-active，xx-leave-active 里面

14.多元素动画

  • 语法：
    • 标签改为 transition-group，
    • 给每个动画标签加上 key，才能识别
    • 其他与单元素动画一致

15.slot插槽属性

Vue.js 实例属性

1.this

  • 默认情况下this都是指的当前Vue实例
    • data的属性和methods的方法都，直接绑定在Vue实例一级下

1.1 this.$nextTick

  • 作用：setTimeout的高级版本，异步操作
    • 从数据改变 到 页面的渲染，是需要时间的，
    • 回调函数会在下一次Dom更新才调用
    • this.$nextTick( 回调函数 )

2.computed计算属性

  • 作用：处理模板中的复杂逻辑

    • 数据需要经过一系列运算得到一个结果（多处要用）时就用计算属性
    • 当你想要对模板数据进行任何复杂逻辑处理，你都应当使用计算属性。

  • 特点

    • 本质上是函数，但可以当属性去渲染
    • 只计算一次，把值缓存起来，直到依赖的值发生变化，它会重新调用计算出来再缓存

Vue.js 组件

1.组件是什么

  • 把某个界面或者说功能封装起来（html css js的综合封装），
    • 组件也是Vue的实例，有自己的data和methods以及生命周期钩子等
    • vue的组件文件是 .vue 文件

2.安装组件依赖

2.1 vue_cli

  • 作用：脚手架，管理项目的工具

2.2 快速原型开发

  • 作用：脚手架启动和新建项目需要时间
    • 快速原型开发提供的，单文件组件服务让我们可以快速的把.vue文件跑起来

2.3 运行.vue文件

3.组件的结构

  • template

    • 写html，只能放一个根标签
    • template标签最终不会出现在界面，它的内容会出现在界面

  • script

    • data必须是一个函数，return一个对象，变量就声明在这个返回的对象里

  • style标签

    • 写样式
    • 引入外部样式
      • 需要在script中，import ‘ 样式路径 ‘，
      • 如果是当前目录，不可以省略./

4.引入其它组件

如何在一个组件中引入其它组件,实现一个组装。

  • 步骤
    1. import 组件名 from ‘ 组件的路径 ‘，路径的文件名可以省略.vue
    2. 在父组件中，加入components对象属性（与data同级）
    3. 在需要用的地方写这个组件名字的标签，例： <son />

5.引入外部插件

  • 示例
    1. 下包：npm i axios
    2. 导包：import 名字 from ‘包名’
    3. 用包

6.父子组件传值

如果A组件中引入了B组件 ，这样我们称A组件为父组件，B为子组件

  • 用法：
    1. 父页面下：
      • 在子组件的标签上，加 ref=”名字”
      • 使用 this.$refs.名字 就能得到子组件对象
      • 此时 this.$refs.名字，就是相当于子组件里面的this
    2. 子页面下：
      • 使用 this.$parent，就能得到父组件对象
  • 其他
    • Vuex是Vue提供的，专门用来处理传值的插件，此处不做扩展

7.父子组件props传值

  • 父组件
    • 在子组件标签上定义一个属性，属性名=属性值
  • 子组件
    • 在子组件内定义一个 props(data同级):[“属性名”]
    • 接受值

Vue.js 生命周期

Vue的生命周期含义：Vue实例从创建到销毁的过程++

官方地址👉：[生命周期图示]

1.beforeCreate

创建之前

  • 作用：它在data和methods的数据与方法挂载到vue实例之前调用
    • beforeCreate 是最早被调用的生命周期钩子
    • 几乎不用

2.created 常用

创建伊始

  • 作用：它在data和methods的数据与方法挂载到vue实例之后调用
    • 此时可以访问到data与methods中的数据与方法
    • 常用来初始化一些数据

3.beforeMount

  • 作用：数据渲染之前调用
    • 此时的dom都仅仅是模板
    • 用的少

4.mounted 常用

  • 作用：渲染之后调用，
    • 要找dom元素，一定在mounted里找
    • 常用

5.beforeUpdate

  • 作用：vue实例里的数据发生改变，但还没渲染到页面
    • 用的少

6.updated

  • 作用：vue实例里的数据发生改变后，并且已经渲染到页面
    • 任意数据发生改变都会触发这个钩子，如果代码写在这略耗性能
    • 用的少

下面两个生命周期用来善后的，例如清除计时器、和window的一些事件监听，以及保存一些重要数据

7.beforeDestroy

  • 组件销毁前（这时候还可以访问到dom和数据）

8.destroyed

  • 组件销毁后（没有完整销毁，还可以访问data中的数据，但是无法访问dom）

Vue中的Css作用域

  • 起步：默认情况下组件的Css样式都是全局的，联动组件通用

    • 如果Style标签加了scoped，则组件样式只给自己以及子组件的根标签使用

  • scoped的原理

    • 给组件样式添加一个唯一的属性选择器
      • 例如：box[ xxx ] ，为组件内 CSS 指定作用域，并且加大了权重
      • 本组件的样式只能
        1. 本组件使用，
        2. 子组件的根标签可以使用，仅限于根标签

  • 细节扩展

    1. 如果父子组件都加了scoped，那么只有子组件的根标签能获得通用Css样式

    2. 如果父子组件都没加scoped，就看各自的优先级权重

    3. 如果又想加scoped，又想设置子组件钟跟标签以外的标签属性

      3.1 在属性前面加 /deep/ 深度选择器

Vue-router 路由

1.路由是什么

  • 作用：路由是一个指向，把路径指向相应的组件地址

    • 根据不同的路径去显示不同的页面

    • 本质是切换组件

2.实现原理

  • 利用锚点：
    1. 锚点也叫 hash，网址 # 号后面的部分
    2. 锚点改变不会跳转页面
    3. 监听锚点改变事件：window.onhashchange
    4. 获取 hash值：window.location.hash
    5. 根据 hash值显示相应的组件
