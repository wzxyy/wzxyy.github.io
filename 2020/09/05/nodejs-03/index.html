搭建静态资源服务器，接受get与post参数

写在开头,get与post的区别

  1. 位置不同

    • get是在url后面拼接 （请求行）
    • post是在请求体中发送 (请求体)

  2. 大小不同

    • get发送数据有大小限制（一般1MB）
    • post发送数据没有大小限制
    • 所以，一般文件上传接口都是post

  3. 速度不同

    • get速度比post快 (其他条件相同的情况下)

  4. 安全性不同

    • post比get安全

    • 一般登录注册都是post

nodejs搭建静态资源服务器

  1. 浏览器中的所有外部资源路径，都会变成网络请求
  2. 一般静态资源(css文件，图片，音视频)使用 url路径拼接的方式来处理

测试1，当资源文件夹在当前js文件的父目录的兄弟目录下，可采用 ’ .. ‘

测试2 当资源文件夹与当前JS文件为兄弟目录时，则可用 ’ 兄弟目录（例如zzz） ’

接收 get 请求参数

  1. 前端发送get请求： 直接在url后面拼接 url?key1=value1&key2=value2

  2. nodejs接收get请求参数

    • 2.1 导入url模块

    • 2.2 调用parse()方法解析参数： url.parse( req.url,true )

    • 2.3 获取解析好的query属性 ： get参数

接收 post 请求参数

  1. 前端发送post

    • 1.1 需要设置请求头
    • 1.2 请求体中发送： xhr.send(key=value)
    • 1.3 数据会被切片成很多数据包，一段一段发送给服务器

  2. nodejs接收post参数

    • 2.1 给req注册data事件 (接收数据包)
    • 2.2 给req注册end事件 (数据包接收完毕)
    • 2.3 在end方法中使用querystring模块解析body
