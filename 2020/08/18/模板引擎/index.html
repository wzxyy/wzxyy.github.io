模板引擎

模板引擎art-template

传送门:https://aui.github.io/art-template/zh-cn/index.html

四个步骤

//1.导包
<script src="./libs/template-web.js"></script>

//2.写模板
//a. 一定要写id属性
//b. 一定要设置type属性， 建议写成 type=text/html
<script id="weather_list" type="text/html">
    <p>{{ data.xx }}</p>
</script>

//3.调用官方API
//第一个参数：模板id  第二个参数：要渲染的js对象  返回值：解析的html字符串
template('weather_list',backData);

//4.渲染解析好的html字符串
document.body.innerHTML += template('weather_list',backData);

模板常用语法

<script id="weather_list" type="text/html">
    <p> {{ data.ganmao }} </p>
    <p> {{ data.wendu + 1 }}</p>
    <p> {{ +data.wendu + 1 }}</p>
    <p> {{ data.wendu > 25 ? '好热':'好冷' }}</p>

    //(2)条件分支
    {{ if data.wendu > 30 }}
        <p>如果条件成立，我才会渲染</p>
        <p>如果条件不成立，我就渲染</p>
    {{ /if }}

    //(3)循环遍历
    {{ each data.forecast}}
    //{{ each data.forecast v i }} 可以自定义字符替换value和index
        <li>
            <span>数组下标： {{ $index }}</span>
            <span>数组元素： {{ $value  }}</span>
        </li>
    {{ /each }}
</script>

易错点

模板引擎易错点总结

  1. 导包

    • 路径错误 ： 固定格式报错 ‘template’ is not defined

  2. 写模板

    • id错误

    • 必须要设置type = text/html

    • 模板语法错误 ： 固定格式报错 ‘告诉你第几行模板语法错误’

    • 对象取值语法错误(单词写错) ： 不报错，但是也不会渲染

  3. 调用官方API

    • id错误 ： 固定错误模板找不到 template not found: Cannot read property ‘value’ of null

    • 第二个参数必须是js对象

  4. 渲染页面

    • 父元素选择器写错了
