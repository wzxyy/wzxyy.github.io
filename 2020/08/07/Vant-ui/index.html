Vant

官网传送门

Vant：轻量、可靠的移动端 Vue 组件库

起步

  • vant ：npm i vant -S

  • 导入组件（导入所有组件的方式）

    import Vant from 'vant'
import 'vant/lib/index.css'
// 使用 vant
Vue.use(Vant)

1.按钮 van-button

按钮支持 default、primary、info、warning、danger 五种类型，

默认为 default。

[其他属性][https://youzan.github.io/vant/#/zh-CN/button]

<van-button type="primary">主要按钮</van-button>
<van-button type="info">信息按钮</van-button>
<van-button type="default">默认按钮</van-button>
<van-button type="warning">警告按钮</van-button>
<van-button type="danger">危险按钮</van-button>

2.单元格 van-cell

如果需要在 cell（单元格之内添加其它的图标），必须使用插槽

  • Cell 可以单独使用，
  • 也可以与 CellGroup 搭配使用，
    • CellGroup 可以为 Cell 提供上下外边框。
<van-cell-group>
  <van-cell title="单元格" value="内容">

    <template #title>
      <div>
        插槽测试1
      </div>
    </template>

    <template #default>
      <div>
        插槽测试2
      </div>
    </template>

</van-cell-group>
  • 注意点：插槽是必须要别人提供的才能用的哦

3.导航栏 van-nav-bar

  • 属性

    • title:中间标题
    • left-text:左边文本内容
    • right-text:右边文本内容

  • 方法

    • @click-left:左边点击事件
    • @click-right:右边点击事件

  • 插槽

    • left:左边,包括左边的图标

    • title:中间标题

    • right:右边

<van-nav-bar title="标题" right-text="按钮"  @click-left="onClickLeft">

  <template #left>
    <div class="iconfont">
      &#xe637;
    </div>
  </template>

</van-nav-bar>

methods:{
    onClickLeft(){window.console.log('xxx')}
}

4.输入框 van-field

  • 属性

    • v-model双向绑定
    • rules定义表单验证规则
    • label:标题
    • name:(类似原生表单提交的name),可定义后用于表单验证与重置
    • type:手机键盘类型(text/tel/digit/number/password)

  • 常用插槽

    • left-icon:自定义输入框头部图标
    • button:自定义输入框尾部按钮
<van-field v-model="number" type="number" label="数字">

  <template #left-icon>
    <div class="iconfont">
      &#xe60b;
    </div>
  </template>

</van-field>

data () {
    return {
      number: ''
    }
  },

5.表单 van-form

  • 属性

    • label-width：标题宽度
    • ref
      • this.$refs.ref值.validata()：表单验证

  • 方法

    • 全局验证
      • this.$refs.ref值.validate().then(() => {}).catch(() => {})
    • 局部验证
      • this.$refs.ref值.validate( “子项name的值” ).then(() => {}).catch(() => {})
      • 就是在 validate 中 加入 子项中自定义的 name 属性值
    • 重置表单验证提示
      • this.$refs.form.resetValidation(‘ 子项name的值 ‘)
      • 不是重置表单哦，重置的是验证提示的文字 QAQ

  • 子项

    • v-model=“ form.xxx ” 绑定数据
    • :rules = “ rules.xxx ” 绑定验证规则
    • name=“xxx” 是局部验证需要的参数
<van-form ref="form"> 
  <van-field
    label="用户名"
    v-model="form.usname"
    :rules="rules.usname"
    name="usname"
  ></van-field>
</van-form>
data(){
    form: {
        usname: ''
      },
     rules: {
        usname: [
          {required: true,message: '不能为空',trigger: 'onChange'},
          {
            validator: value => {
              const _reg = xxx
              return _reg.test(value)
            },
            message: '请输入正确的手机号',
            trigger: 'onChange'
          }
        ],
    }    
}

6.轻提示 van-toast

提示用户时,最好有一些美观的提示效果,提升用户体验,这里可以使用toast组件来实现

van - toast

  • 核心
    • this.$toast
  • 清理提示
    • this.$toast.clear()
  • 常用参数
    • type: 可选 loading，success，fail
    • duration: 时长,毫秒为单位，如果写0表示不消失,
    • overlay: 是否显示背景层
    • forbidClick : 是否禁止背景点击
    • message: 消息内容
    • loadingType：自定义图标，可选
this.$toast.success('成功')
this.$toast.fail('失败')
this.$toast.loading('加载')

his.$toast.loading('加载')
  this.$toast.loading({
    loadingType: 'spinner',
    duration: 2000,
    message: '加载'
})
