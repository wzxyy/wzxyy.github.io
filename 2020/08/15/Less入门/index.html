less

记录一下less，方便自己快速查阅

写在开头

Less是一门CSS预处理语言，它扩展了CSS语言，增加了变量，Mixin，函数等特性，使CSS更易维护和扩展。

详细了解请戳我 👉 Less中文网

安装

通过npm安装

全局安装

npm install -g less

查看版本号

lessc -v

使用方法

创建一个test.less文件

cd到当前目录，命令窗口: lessc test.less test.css

vsCode安装

扩展商店 👉 Easy LESS

语法

变量

@width: 10px;
@color: zz;
@zz:green;
#box{ 
    widht:@width;
    color:@@color // 相当于@zz
}

此时#box的宽为10px; 颜色是green

混合(Mixins)

.testClass { width:10px }
#box{ .testClass(); }

此时#box拥有.testClass的属性

你也可以使用 #ids 作为 mixin 使用

嵌套写法

#box{
    width:10px;
    &:hover{}
    p{
        width:5px;
    }
}

相当于 #box:hover{};

运算

@widht: 2px + 8px; // 结果是10px
@test: @widht + @widht;// 结果是20px

@color: #224488 / 2; // 结果是#112244
background-color: #112244 + #111; // 结果是#223355

支持对任何，数字，颜色，变量。加减乘除。

以左边数的单位为准，如果失去意义就忽略单位。

calc() 特例

@var: 50px/2;
width: calc(50% + @var);  // 结果是 calc(50% + 25px)

为了保持和原生的一致，只会计算变量函数的值，calc里面的不动

css原生calc()点我，用于动态计算长度值

@的规则嵌套和冒泡

.component {
  width: 300px;
  @media (min-width: 768px) {
    width: 600px;
    @media  (min-resolution: 192dpi) {
      background-image: url(/img/retina2x.png);
    }
  }
  @media (min-width: 1280px) {
    width: 800px;
  }
}

编译成css的结果

.component {
  width: 300px;
}
@media (min-width: 768px) {
  .component {
    width: 600px;
  }
}
@media (min-width: 768px) and (min-resolution: 192dpi) {
  .component {
    background-image: url(/img/retina2x.png);
  }
}
@media (min-width: 1280px) {
  .component {
    width: 800px;
  }
}

@ 规则（例如 @media 或 @supports）可以与选择器以相同的方式进行嵌套。@ 规则会被放在前面，同一规则集中的其它元素的相对顺序保持不变。这叫做冒泡（bubbling）。

转义

@min768: (min-width: 768px);
.element {
  @media @min768 {
    font-size: 1.2rem;
  }
}
/*
@media (min-width: 768px) {
  .element {
    font-size: 1.2rem;
  }
}
*/

转义（Escaping）允许你使用任意字符串作为属性或变量值。任何 ~"anything" 或 ~'anything' 形式的内容都将按原样输出，除非 interpolation。

可变插值(Variable Interpolation)

@images: "../img";

body {
  color: #444;
  background: url("@{images}/bg.png");
}

image-size，image-width，image-height

从文件获取的图像尺寸。

image-size("file.png");//输出：10px 10px
image-width("file.png");//10px
image-height("file.png");//10px

命名空间访问符

有点像对象，封装的感觉

#bundle() {
  .button {
    display: block;
    border: 1px solid black;
    background-color: grey;
    &:hover {
      background-color: white;
    }
  }
  .tab { ... }
  .citation { ... }
}

现在，如果我们希望把 .button 类混合到 #header a 中，我们可以这样做：

#header a {
  color: orange;
  #bundle.button();  // 还可以书写为 #bundle > .button 形式
}

注意：如果不希望它们出现在输出的 CSS 中，例如 #bundle .tab，请将 () 附加到命名空间（例如 #bundle()）后面。

映射

像对象取值一样

#colors() {
  primary: blue;
  secondary: green;
}

.button {
  color: #colors[primary];
  border: 1px solid #colors[secondary];
}

输出符合预期：

.button {
  color: blue;
  border: 1px solid green;
}

作用域

@var: red;

#page {
  @var: white;
  #header {
    color: @var; // white
  }
}

与 CSS 自定义属性一样，混合（mixin）和变量的定义不必在引用之前事先定义。因此，下面的 Less 代码示例和上面的代码示例是相同的：

@var: red;

#page {
  #header {
    color: @var; // white
  }
  @var: white;
}

注释

/* 一个块注释
 * style comment! */
@var: red;

// 这一行被注释掉了！
@var: white;

导入

“导入”的工作方式和你预期的一样。你可以导入一个 .less 文件，此文件中的所有变量就可以全部使用了。如果导入的文件是 .less 扩展名，则可以将扩展名省略掉：

@import "library"; // library.less
@import "typo.css";
